## 1. Overview & links

The desktop redesign evolves the existing shell into a shippable **solo local-desktop 0xCopilot app**:
the 6-destination shell in the v2 "quiet" design language, the production chat interaction layer
consolidated into `packages/chat-surface` (single source of truth), **Run** as the flagship cockpit,
a first-class solo **Settings** surface, and team features gated behind `ENTERPRISE_DEPLOYMENT_PROFILE`.

## 4. Consistency report

### What holds

- **ID scheme consistent.** Every phase uses `US-<phase>.<n>`, `FR-<phase>.<n>`, `PR-<phase>.<n>` per
  `_TEMPLATE.md`. PR ids are phase-scoped and globally unique (no `PR-x.y` collisions across phases).
- **DAG integrity.** No dependency cycle exists intra- or cross-phase. Every cross-phase edge points from an
  earlier phase to a later one (0→1, 0→2, 0→5, 1→3, 2→3, 2→4, 2→5, 2→6, 3→4, 3→6, 4→6, 5→6). Sequence is valid.
- **SSOT adherence (mostly).** `chat-surface` is the canonical home for interaction components (Phase 1),
  destinations registry (`shell/destinations.ts`), Run cockpit (`destinations/run/`), and Settings
  (`settings/`); `packages/design-system/src/styles.css` is the only token source; `packages/api-types` owns
  wire contracts (Phase 4 FR-4.33 forbids re-declaring types in chat-surface). Each phase re-affirms the
  ADR (a): one component, two consumers, `apps/frontend` as re-export shim.
- **App/package boundary adherence.** Every phase restates and tests the `CLAUDE.md` hard rule — no
  `apps/*` imports another app's `src/`; `chat-surface` stays framework-agnostic (ports only, ESLint
  substrate guard from Phase 0E). Desktop consumes `@0x-copilot/chat-surface`, never `apps/frontend`.
- **Test-runner conventions consistent.** All TS packages/apps use **vitest** via
  `npm run test --workspace <pkg>`; typecheck via `npm run typecheck --workspace <pkg>`; Python (where
  touched) via the owning service `.venv` **pytest**; live desktop smoke per `apps/desktop/SMOKE.md`. Every
  phase carries an FR→test map and a web-regression guard.
- **Coverage vs PLAN scope.** All PLAN §8 letter-items map to PRs with no dropped scope: 0A–0E → PR-0.1–0.6;
  1A–1F → PR-1.1–1.8 (composer split 1B→PR-1.2/1.3, plus PR-1.8 finalize); 2A–2E → PR-2.1–2.6; 3A–3G →
  PR-3.1–3.11; 4A–4E → PR-4.1–4.11; 5A–5E → PR-5.1–5.9; 6A–6E → PR-6.1–6.9. Each finer split is justified
  by the ≤~1000 LOC rule; no phase adds scope outside its PLAN sketch.
