# AC10 ‚Äî Hardening, Rollout & Backout (release gate)

Status: **evidence bundle** ¬∑ Owner: Desktop + Agent Runtime ¬∑ Last updated: 2026-07-18

AC10 is not a feature; it is the **gate** that lets the desktop file-native store and the AC5‚ÄìAC9 capabilities move from opt-in (`COPILOT_DESKTOP_FILE_STORE_V1`, `RUNTIME_STORE_BACKEND=file` under `single_user_desktop`) to default-on. It exists to answer one question honestly: _is the evidence sufficient to flip the default?_

This document follows the repo compliance rule (`CLAUDE.md` ‚Üí Compliance Reviews): **a control counts only when code, config, tests, and docs all support it**, and **product controls are separated from deployment controls**. Rows below are marked `evidenced-in-repo` (code + tests + docs present on `feat/desktop-redesign`) or `deployment-evidence-outstanding` (requires a packaged build / host run not reproducible in unit CI).

## 1. Verdict

**Not yet default-on.** All _product-control_ evidence is complete, merged, and tested in-repo. The remaining gate is _deployment evidence_: a packaged cross-platform smoke + a data-preserving backout drill on real macOS arm64/x64 and Windows x64 builds. Until that runs, the capabilities stay opt-in behind their flags. No default was changed by any AC1‚ÄìAC10 PR.

## 4. Deployment evidence ‚Äî OUTSTANDING (blocks default-on)

These are **deployment controls** and cannot be discharged by unit CI. They must run on packaged builds before the default flips:

1. **Packaged cross-platform smoke** ‚Äî macOS arm64 + x64 and Windows x64 packaged boot (embedded PostgreSQL + three supervised services), one real chat/run/approval end-to-end on `RUNTIME_STORE_BACKEND=file`. (`apps/desktop/SMOKE.md` is the harness.)
2. **Data-preserving backout drill** ‚Äî migrate postgres‚Üífile, run on file, then flip back: prove no conversation/run/event loss in either direction (the migration is one-way by design, so backout = keep the postgres source until the smoke passes; document the retention window).
3. **Retention-expiry on a packaged build** ‚Äî quota rejection + age-sweep exercised against a real on-disk store, not a tmp fixture.
4. **Process-leak check** ‚Äî no orphaned browser contexts / sandbox processes / loopback listeners after quit + crash.
5. **Windows path-security parity** ‚Äî the adversarial path corpus (devices/ADS, junction/reparse, mount swap, TOCTOU) on Windows; macOS is covered in-repo (#66) but the Windows native-FS helper (#13) needs the packaged run.

## 5. Known deferred gaps (non-blocking, tracked)

- Hard **1 MiB payload rejection** (AC2 rule 4) ‚Äî the adversarial matrix asserts no-collateral-corruption, not rejection; enforcement not yet in code.
- Monty: factory threading of the run's model-visible toolset into `external_tools_by_name` (seam in place, defaults pure-compute) + CAS dedup on `(run_id, session_id, invocation_index)` for worker-crash double-dispatch.
- Browser: concrete worker‚Üîmain approval-port transport (fail-closed seam present); `browser_upload` (needs an AC5 object-ref grant); ephemeral-profile consent UX.
- AC7 sandbox: snapshot-in/patch-out over AC4/AC5, deny-all egress, short-lived secret refs, durable reaper ‚Äî only the execute-only tool is merged.
- Needs-repair **UX**: `GET /v1/agent/store/health` route ‚Üí facade proxy ‚Üí `api-types` DTO ‚Üí conversation-list banner (data/status API done in #69; UI deferred).

## 6. Gate decision rule

Flip the default **only** when ¬ß2 stays green on the target tip **and** every ¬ß4 item has recorded evidence (packaged smoke logs + backout drill result). ¬ß5 gaps are acceptable at default-on **if** each is either flag-gated off or has a stated, honest limitation in the user-facing disclosure. Until then: opt-in only.
